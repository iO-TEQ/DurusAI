# HMI LABEL COMPONENT FOR DURO CONTROLLER

- Label is one of the components available for the hmi.

Description:
- The Label component displays static or dynamic text on the HMI. It can be styled for appearance, animated based on tag values, and respond to user interactions via events. Labels support positioning, sizing, rotation, visibility, and z-order control, and can be bound to tags for live updates.

Core Properties:
- id: Unique identifier for the component (prefixed by `lbl_`).
- viewId: Identifier of the parent view/screen.
- type / typeAbbr: Component type (`label`) and abbreviation (`lbl_`).
- comptName: Human-readable name of the component.
- visibility: Initial visibility state (`true` or `false`).
- x, y: Position coordinates of the label in the view.
- w, h: Width and height of the label.
- zIndex: Stacking order relative to other components.
- rotationAngle: Rotation in degrees applied to the label.
- sizeMode: Sizing mode behavior (e.g., `zoom`).

Config:
- placeholder: Default text displayed when no tag/content is bound.
- buttonMode: Treats the label like an interactive button when `true`. 
- style: Visual styling properties using CSS-like keys:
    - justify-content: Horizontal alignment of the content within the label. Options: `flex-start`, `center`, `flex-end`.
    - align-items: Vertical alignment of the content within the label. Options: `flex-start`, `center`, `flex-end`.
    - fontFamily: Font family used for the text. Options: `None`, `Arial`, `Brush Script MT`, `Copperplate`, `Courier New`, `Garamond`.
    - font-weight: Font weight. Options: `None`, `Bolder`, `Normal`, `Lighter`.
    - fontSize.px: Font size in pixels.
    - color: Text color (hex or rgba).
    - backgroundColor: Background fill color.
    - borderRadius.px: Corner radius in pixels.
    - border: Border style string (e.g., `solid #000000 2px`).
    - box-shadow: Shadow definition (e.g., `5px 5px 10px 0px #000000`).
    - feedback-fx: click animation. buttonMode must be true for animation to work. (e.g., `scale(0.9) #1800ff 1s`)

Animation:
- Allows dynamic property changes driven by tag values or expressions:
    - backgroundColor: Expression to set background color conditionally.
    - border: Expression to set border color/style conditionally.
    - text: String or tag binding for the labelâ€™s displayed text.
    - visibility: Boolean expression controlling visibility (e.g., `numberTag == 1`).
    - color: Expression to set text color conditionally.
- Expressions support conditional logic via `if(condition, valueIfTrue, valueIfFalse)` and tag comparisons.

Events:
- click: Action or script to execute on user click. When `buttonMode` is enabled, the label behaves like a button and can trigger interactions.

Usage Notes:
- Bind `animation.text` to a tag (e.g., `textTag`) to show live values.
- Use `Animation` expressions to reflect state changes visually (e.g., alarm colors).
- Apply `style` for consistent look and feel; prefer `.px` suffixed properties for pixel-based values.
- Keep `placeholder` meaningful for design-time clarity when no tag is present.
